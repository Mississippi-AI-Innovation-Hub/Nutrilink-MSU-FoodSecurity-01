# Limitations

NutriLink was developed as a proof-of-concept and is not production-ready.

## Known Limitations

- The PoC focused only on Hinds County, Mississippi.
- The GUI was lightweight and not fully production refined.
- Response quality through the API-driven GUI was less consistent than direct Bedrock Playground testing.
- No production MDHS systems were integrated.
- No real citizen data was collected, stored, or processed.
- No authentication, authorization, monitoring, or production security hardening was implemented.
- Advanced HER/SWAP food classification and recommendation logic was only partially implemented.
- No formal quantitative benchmark was completed.
- The project depended on AWS sandbox permissions and available service access.

## Out of Scope

- Production deployment.
- Statewide deployment across all Mississippi counties.
- Predictive forecasting.
- Autonomous or binding agency decisions.
- Protected data integrations.
- Public launch or unsupported external access.

## Disclaimer

This repository contains code and supporting materials developed as part of a Mississippi Artificial Intelligence Innovation Hub Proof of Concept project. The contents are provided for prototype demonstration purposes. They are not production ready by default and may include simplified workflows, incomplete security guardrails, placeholder integrations, or reduced controls appropriate only for a Proof-of-Concept environment.
